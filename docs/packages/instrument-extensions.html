Instrument Extensions¶

Tokens¶

  • Daml.Finance.Interface.Instrument.Base

    This package contains the interface for a basic instrument. It contains the following modules:

    • Instrument: Base interface for an instrument. This interface does not define any lifecycling logic.

  • Daml.Finance.Instrument.Standard

    This package contains the implementation of a basic instrument. It contains the following module:

    • Instrument: Standard implementation of an Instrument which does not define any lifecycling logic.

    Check out the Transfer tutorial for an example on how to create a base instrument and use it for a transfer.

Bonds¶

  • Daml.Finance.Interface.Instrument.Bond

    This package contains the interface for different bond types, defined in the following instrument modules:

    • FixedRate: Interface that allows implementing templates to create fixed rate bonds.
    • FloatingRate: Interface that allows implementing templates to create floating rate bonds.
    • InflationLinked: Interface that allows implementing templates to create inflation linked bonds.
    • ZeroCoupon: Interface that allows implementing templates to create zero coupon bonds.

  • Daml.Finance.Instrument.Bond

    This package contains the implementation of different bond types, defined in the following instrument modules:

    • FixedRate: This template models a fixed rate bond. It pays a fixed coupon rate at the end of every coupon period.
    • FloatingRate: This template models a floating rate bond. It pays a floating coupon rate at the end of every coupon period.
    • InflationLinked: This template models an inflation linked bond. It pays an inflation adjusted coupon at the end of every coupon period.
    • ZeroCoupon: This template models a zero coupon bond. It does not pay any coupons, only the redemption amount at maturity.

    Check out the tutorial on How to use the Bond extension package for a description how to use the bond extension in practice. There is also the tutorial How to implement a Contingent Claims-based instrument, which describes how the claims are defined and how the lifecycle interface is implemented for bonds.

Swaps¶

  • Daml.Finance.Interface.Instrument.Swap

    This package contains the interface for different types of swaps. It contains the following instrument modules:

    • CreditDefaultSwap: Interface that allows implementing templates to create credit default swaps.
    • CurrencySwap: Interface that allows implementing templates to create currency swaps.
    • ForeignExchange: Interface that allows implementing templates to create foreign exchange (FX) swaps.
    • InterestRate: Interface that allows implementing templates to create interest rate swaps.

  • Daml.Finance.Instrument.Swap

    This package contains the implementation of different types of swaps. It contains the following instrument modules:

    • CreditDefaultSwap: This template models a credit default swap. It pays a fix rate vs 1-recoveryRate (in case of a credit default event).
    • CurrencySwap: This template models a currency swap. It pays a fix vs fix rate (in different currencies) at the end of every payment period.
    • ForeignExchange: This template models a foreign exchange (FX) swap. It has two legs: an initial FX transaction and a final FX transaction.
    • InterestRate: This template models an interest rate swap. It pays a fix vs floating interest rate at the end of every payment period.

Equity¶

  • Daml.Finance.Interface.Instrument.Equity

    This package contains the interface for equities. It has the following modules:

    • Factory: Interface that allows implementing templates to create equity instruments.
    • Instrument: Interface for a generic equity instrument.

  • Daml.Finance.Instrument.Equity

    This package contains the implementation for equities. It has the following modules:

    • Factory: Factory template for instrument creation.
    • Instrument: Instrument representing a common stock.

    For a detailed explanation of the equity extension, check out the src/test/daml/Daml/Finance/Instrument/Equity/Test folder. It demonstrates how to originate an equity instrument, how to create and lifecycle a cash dividend, and how to handle corporate actions like mergers and stock splits.

Generic¶

  • Daml.Finance.Interface.Instrument.Generic

    This package contains the interface and types required for generic instruments using Contingent Claims, including lifecycling logic. It contains the following modules:

    • Election: Interface implemented by templates that represents a (claim-based) election.
    • Factory: Interface that allows implementing templates to create generic instruments.

  • Daml.Finance.Instrument.Generic

    This package contains the implementation and types required for generic instruments, including lifecycling logic. It contains the following modules:

    • Election: Implementation of Election (e.g. the exercise of an option) and ElectionFactory (to delegate the right to create Elections).
    • Factory: Factory template for generic instrument creation.
    • Instrument: An instrument representing a generic payoff, modelled using Contingent Claims.
    • Util: Utility functions related to creating Contingent Claims for bonds/swaps including lifecycling logic.

    The tutorial How to use the Derivative extension to model generic instruments describes how a payoff is defined using Contingent Claims in practice.
